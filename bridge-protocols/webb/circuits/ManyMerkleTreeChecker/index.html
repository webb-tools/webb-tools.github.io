    ManyMerkleTreeChecker¶

    template ManyMerkleTreeChecker(levels, length)

    Inputs¶

    1. leaf (corresponds to a commitment on an anchor Merkle tree)
    2. pathElements[levels]
    3. pathIndices[levels]
    4. roots[length]
    5. diffs[length]

    Outputs¶

    None

    Sub-Gadgets¶

    1. DualMux(). See:
    2. HashLeftRight() which is a two input Poseidon hasher.
    3. SetMembership(length). See:

    Functionality/How it Works¶

    Uses HashLeftRight()to compute Merkle root from pathElements[levels] and the leaf.

    The DualMux() gadget is necessary because depending on whether the Merkle proof element is on the left or right side (as indicated by pathIndices[levels]), the inputs to HashLeftRight() have to be re-ordered. This is because Poseidon(left, right) does not equal Poseidon(right, left).

    Uses the SetMembership(length) gadget to check whether the computed root is in roots[length].
